Raspberry Pi – Erste Schritte

13 Dezember 2015 in Raspberry Pi

Ziel

Um ein wenig mit dem Raspberry Pi herumspielen zu können ist das Ziel dieses Beitrags, erst einmal den Raspi so weit zu konfigurieren, dass wir später über eine SSH-Verbindung auf ihn zugreifen können. Dazu installieren wir ein Betriebssystem, nehmen Konfigurationen vor, aktualisieren die Firmware und verbinden uns mit SSH auf den Raspi.

In weiteren Beiträgen folgt dann die Ansteuerung der GPIO Schnittstelle, um im ersten Schritt ein paar Leuchtdioden zum Leuchten zu bringen und später dann auch Servos und Motoren anzusteuern. Auch die Kamera des Pi soll in einem Beitrag zum Leben erweckt werden.

Betriebssystem installieren

Am einfachsten geht die Installation des Betriebssystems mittels NOOPS Lite, die Installationsroutine kann hier heruntergeladen werden. Den Inhalt der ZIP-Datei auf eine leere SD-Karte mit FAT-Dateisystem kopieren, in den Raspberry Pi einlegen und den Raspi starten, indem er über die Stromversorgung angeschlossen wird. Der vorherige Anschluss eines Monitors über die HDMI Schnittstelle macht noch Sinn, da man ansonsten nicht viel sieht :-). Und natürlich sollte eine Tastatur und eine Maus auch nicht fehlen, sonnst klappt es mit der Bedienung nicht ;-).

Voraussetzung für die Installation über NOOPS Light ist, dass der Raspi eine Verbindung ins Internet hat, da NOOPS Light während der Installation Dateien herunterlädt. Dazu einfach eine Verbindung zum Router über ein Ethernetkabel herstellen. Bei mir kam dann eine Fehlermeldung, dass keine Netzverbindung besteht. Nach der Auswahl des Betriebssystems Raspbian wurde aber die Internetverbindung erkannt.

Es werden dann ca. 3,2 GB an Daten heruntergeladen. Die SD Karte sollte also mindestens eine Größe von 8 GB haben, um später auch noch Software installieren zu können.

Raspbian starten

Nachdem die Installation durchgeführt wurde startet der Raspi neu, direkt in den grafischen Desktop von Raspbian. Über Menu/Preferences/Raspberry Pi Configuration kann man nun z.B. die Sprache, die Zeitzone  oder das Tastaturlayout einstellen. Da ich den grafischen Desktop nicht verwenden möchte, stelle ich hier nun nur unter System die Einstellung Boot auf To CLI (Command Line Interface = Kommandozeile oder auch Konsole) und mache die anderen Konfigurationen später über die Konsole. Nach Auswahl von _OK _kommt die Nachfrage, ob nun neu gebootet werden darf. Hier also YES auswählen.

Konfiguration über die Konsole

Zeitzone und Tastaturlayout

Dach dem der Raspi nun neu gebootet wurde, gelangen wir direkt in die Konsole:

pi@raspberrypi:~ $ _

Die Konfiguration über die Kommandozeile kann nun über sudo raspi-config gestartet werden (den Bindestrich erreicht man über das ß). Als erstes ändern wir über den Punkt 5 Internationalisation Options die Zeitzone (in meinem Fall Europe/Berlin) und vor allem das Layout der Tastatur, falls man mal eine angeschlossen hat. Meine Cherry DW 5000 funktioniert gut mit der Einstellung _Generic 105-key (Intl) PC_ und dem Keyboard layout über other/German/German. Bei den restlichen Abfragen habe ich die Default-Werte übernommen. Die Sprache habe ich bewusst auf Englisch gelassen, da ih keine Lust auf irgend welche mißverständlichen Übersetzungen habe…

SSH

Am besten erreicht man später den Raspi über eine SSH Verbindung von einem anderen Computer aus. So spart man sich die Tastatur, Maus und den Monitor für den Raspi. Das ist natürlich keine Lösung für alle, die den grafischen Desktop von Raspbian verwenden wollen.

Normalerweise ist der SSH Server bereits eingeschaltet. Dies kann man in der Konfiguration unter Advanced Options/SSH kontrollieren, hier sollte Enable vorbelegt sein.

Hostname

Unter Advanced Options/Hostname kann der Name des Raspi geändert werden, der Standardname lautet raspberrypi. Ich habe hier einen neuen Namen vergeben, da ich meine zwei Raspis gerne unterscheiden möchte.

Die Frage nach dem Reboot dann mit YES bestätigen.

Update der Firmware

Um alles auf dem aktuellsten Stand zu haben, kann man auch noch direkt ein Firmware Update durchführen.

Die Firmware kann am einfachsten mit diesem Script von Liam McLoughlin aktualisiert werden. Dazu laden wir es erst einmal von GitHub herunter und legen es im Dateisystem unter /usr/bin ab. Dies geschieht über den Befehl

sudo wget https://raw.github.com/Hexxeh/rpi-update/master/rpi-update -O /usr/bin/rpi-update

Nun  das Script noch über sudo chmod +x /usr/bin/rpi-updateausführbar machen und das Update mittels sudo rpi-updatestarten. Vorher kann man noch, wenn man möchte, die Version über uname -rabfragen.  Bei mir lautet die Version aktuell 4.1.13-v7+. Der nötige Reboot kann dann über den Befehl rebootausgeführt werden. Eine anschließende Versionsabfrage ergab bei mir 4.1.14-v7+.

Verbindung über SSH

Um eine Verbindung über SSH herstellen zu können, benötigen wir die IP-Adresse des Raspi. Über den Befehl ifconfigerhalten wir Informationen über das Netzwerk-Interface: 


  eth0    Link encap:Ethernet  HWaddr b8:27:ee:63:ce:8a
          inet addr:192.168.178.41  Bcast:192.168.178.255  Mask:255.255.255.0
          inet6 addr: fe80::851a:fd6d:f047:d79a/64 Scope:Link
          UP BROADCAST RUNNING MULTICAST  MTU:1500  Metric:1
          RX packets:506 errors:0 dropped:0 overruns:0 frame:0
          TX packets:146 errors:0 dropped:0 overruns:0 carrier:0
          collisions:0 txqueuelen:1000
          RX bytes:82012 (80.0 KiB)  TX bytes:18203 (17.7 KiB)

Hinter _inet addr:_ in Zeile 2 steht die gesuchte IP-Adresse, welche wir notieren.

Statische IP-Adresse

Da der Rapsi normalerweise vom Router über DHCP immer eine neue Adresse zugewiesen bekommt, ist es besser, in den Einstellungen des Routers eine statische IP-Adresse für den Raspi zu vergeben, da man ansonsten immer wieder die IP ermitteln muss. Wie das z.B. bei einer FritzBox geht, wird hier beschrieben.

Nun können wir die Tastatur, die Maus und den Monitor vom Raspi entfernen.

Von einem anderen Computer im selben Netzwerk und dessen Konsole aus verbinden wir uns nun über SSH auf den Raspi über den Befehl ssh pi@192.168.178.41. Das Standard-Passwort lautet raspberry, welches man am besten direkt über den Befehl passwd pi ändert. Anschließend erscheint die Konsole des Raspi und wir haben es geschafft!

SSH konfigurieren

Die Verbindung kann noch einfacher erfolgen, indem man SSH entsprechend konfiguriert. Ich erkläre das nun im Folgenden anhand der Konfiguration auf einem Mac.

In der Konsole wechseln wir in das Verzeichnis .ssh, welches im Homeverzeichnis des angemeldeten Users liegt: cd ~/.ssh. Hier öffnen wir mit open config die Konfigurationsdatei, welche anschließend im Standard-Editor angezeigt wird.

Existiert das Verzeichnis nicht, so kann es mit mkdir .ssh && chmod 700 .ssh angelegt und mit den nötigen Berechtigungen versehen werden. Existiert die Datei nicht, kann sie über den Befehl touch config angelegt werden.

Die Datei kann nun z.B. über einen Eintrag wie 


  Host raspberrypi
  HostName 192.168.178.41
  User pi
erweitert werden. Wobei natürlich die IP-Adresse auf die des eigenen Raspi angepasst werden muss sowie auch der Hostname, falls man diesen geändert hat. Nach dem Speichern kann man den Raspi nun sehr einfach über den Befehl ssh raspberrypierreichen.

Raspberry Pi ausschalten

Will man den Raspberry Pi ausschalten, so sollte man das Betriebssystem vorher über den Befehl sudo shutdown herunterfahren, warten bis es tatsächlich unten ist  und erst dann die Stromversorgung entfernen.
